Presentacion tecnica

CRUD por consola con arquitectura modular y TypeScript estricto

Presentado por la celula Kernel: una app inspirada en Crunchyroll para gestionar Users, Categories y Series. El objetivo no fue solo que corra, sino que sea explicable, mantenible y bien tipada.

CLI en espanol Zero any Decorador activo Clean layering Sesion por rol

01

Arquitectura real del proyecto

Capas

  • domain: entidades
  • application: reglas de negocio
  • infrastructure: tablas en memoria
  • presentation: controllers + views CLI
  • shared: decoradores, errores y utils

Direccion de dependencias

  • presentation -> application
  • application -> domain + infrastructure/database
  • shared transversal

03

Temas investigados y aplicacion en codigo

Clases en TypeScript

  • Constructores y entidades por dominio
  • public, private, protected
  • Herencia con extends
  • Tipos del dominio (ejemplo: UserProps)
  • Propiedades readonly

Namespaces

  • No se usan namespaces
  • Se usa modularidad ES con import/export
  • Alineado con TypeScript moderno en Node
  • Soportado por moduleResolution: "bundler"
Clase + readonly src/domain/entities/User.ts
Herencia + abstract src/presentation/controllers/BaseController.ts

Plan de demo en sustentacion

Comandos de validacion

npx tsc --noEmit
bun run src/index.ts

Flujo recomendado

  1. Seleccionar usuario de sesion (ADMIN o USER)
  2. Crear categoria y serie
  3. Probar validaciones de duplicado (categoria/serie)
  4. Probar borrado de serie y categoria
  5. Probar operaciones admin de users (12-15)
